# 🚀 Deployment Guide - Electron App with Cloudflare R2 Auto-Updates

Complete guide to build and deploy your BG Remover desktop app with automatic updates using Cloudflare R2.

---

## 📦 Prerequisites

### 1. Install Dependencies

```bash
cd "d:\background-remover-master\bg remover desktop"
npm install
```

**New packages added:**
- `electron-updater` - Auto-update functionality
- `electron-store` - Persistent encrypted storage
- `javascript-obfuscator` - Code protection

You need icons in these formats:

**Windows:**
- `assets/icon.ico` (256x256 or multi-size)

**macOS:**
- `assets/icon.icns` (512x512@2x recommended)

**Linux:**
- `assets/icon.png` (512x512 PNG)

**Quick icon generation:**
Use online tools like:
- https://www.icoconverter.com/
- https://cloudconvert.com/png-to-icns
- Or use `electron-icon-builder` npm package

## 🏗️ Building the App

### Build for Windows (on Windows)

```bash
npm run build:win
```

**Output:**
- `dist/BG Remover Setup 1.0.0.exe` - NSIS installer
- `dist/win-unpacked/` - Unpacked files

---

### Build for macOS (on Mac)

```bash

```

**Output:**
- `dist/BG Remover-1.0.0.dmg` - DMG installer (Intel)
- `dist/BG Remover-1.0.0-arm64.dmg` - DMG installer (Apple Silicon)

**Note:** macOS builds require code signing for auto-updates to work properly. For testing, you can disable verification (already set in package.json).

---

### Build for Both Platforms

```bash
npm run release
```

This builds for both Windows and macOS (requires macOS to build Mac apps).

## ☁️ Setup Cloudflare R2 for Auto-Updates

### Step 1: Create R2 Bucket

1. Log in to Cloudflare Dashboard: https://dash.cloudflare.com/
2. Go to **R2 Object Storage**
3. Click **Create Bucket**
4. Bucket name: `sallulabs-app-updates`
5. Location: Automatic
6. Click **Create Bucket**

---

### Step 2: Enable Public Access

1. In your bucket, go to **Settings**
2. Scroll to **Public Access**
3. Click **Allow Access**
4. Click **Add Custom Domain**
5. Enter: `updates.sallulabs.com`
6. Follow DNS setup instructions (add CNAME in Cloudflare DNS)

**Cloudflare DNS Setup:**
```
Type: CNAME
Name: updates
Target: <YOUR-BUCKET>.r2.cloudflarestorage.com
Proxy: ON (orange cloud)
```

---

### Step 3: Upload Build Files

After building your app, upload these files to R2:

**For Windows:**
```
BG Remover Setup 1.0.0.exe
latest.yml
```

**For macOS:**
```
BG Remover-1.0.0.dmg
BG Remover-1.0.0-arm64.dmg
BG Remover-1.0.0-mac.zip (for auto-updates)
BG Remover-1.0.0-arm64-mac.zip (for auto-updates)
latest-mac.yml
```

**Upload via Cloudflare Dashboard:**
1. Open your bucket
2. Click **Upload**
3. Drag and drop files from `dist/` folder

**Upload via Wrangler CLI (recommended for automation):**

```bash
# Install Wrangler
npm install -g wrangler

# Login
wrangler login

# Upload files
wrangler r2 object put sallulabs-app-updates/BG Remover Setup 1.0.0.exe --file="dist/BG Remover Setup 1.0.0.exe"
wrangler r2 object put sallulabs-app-updates/latest.yml --file="dist/latest.yml"
```

---

### Step 4: Set Correct MIME Types

Ensure files have correct content types:

- `.exe` → `application/octet-stream`
- `.dmg` → `application/x-apple-diskimage`
- `.zip` → `application/zip`
- `.yml` → `text/yaml`

R2 usually auto-detects, but you can set manually if needed.

## 🔄 Auto-Update Flow

### How It Works

1. **App starts** → Checks `https://updates.sallulabs.com/latest.yml`
2. **Compares versions** → If newer version available, shows notification
3. **User clicks "Update"** → Downloads new version
4. **Download complete** → Installs and restarts app

---

### Update Files Structure

**latest.yml (Windows):**
```yaml
version: 1.0.1
files:
  - url: BG Remover Setup 1.0.1.exe
    sha512: <hash>
    size: 123456789
path: BG Remover Setup 1.0.1.exe
sha512: <hash>
releaseDate: '2024-01-31T10:00:00.000Z'
```

**latest-mac.yml (macOS):**
```yaml
version: 1.0.1
files:
  - url: BG Remover-1.0.1-mac.zip
    sha512: <hash>
    size: 123456789
  - url: BG Remover-1.0.1-arm64-mac.zip
    sha512: <hash>
    size: 123456789
path: BG Remover-1.0.1-mac.zip
sha512: <hash>
releaseDate: '2024-01-31T10:00:00.000Z'
```

These files are auto-generated by electron-builder.

## 🔒 Code Signing (Optional but Recommended)

### Windows Code Signing

**Why:** Prevents "Unknown Publisher" warnings

**Requirements:**
- Code signing certificate (from DigiCert, Sectigo, etc.)
- Cost: ~$200-400/year

**Setup:**
```json
// package.json
"win": {
  "certificateFile": "path/to/cert.pfx",
  "certificatePassword": "your-password",
  "sign": "./sign.js"
}
```

---

### macOS Code Signing

**Why:** Required for notarization and Gatekeeper

**Requirements:**
- Apple Developer Account ($99/year)
- Developer ID certificate

**Setup:**
```json
// package.json
"mac": {
  "identity": "Developer ID Application: Your Name (TEAM_ID)",
  "hardenedRuntime": true,
  "gatekeeperAssess": false,
  "entitlements": "build/entitlements.mac.plist",
  "entitlementsInherit": "build/entitlements.mac.plist"
}
```

**For now:** Code signing is disabled for testing. Add it before public release.

## 🐛 Troubleshooting

### Update Check Fails

**Check:**
1. Is `https://updates.sallulabs.com` accessible?
2. Does `latest.yml` exist in R2?
3. Are CORS headers set correctly?

**Fix CORS (if needed):**
```javascript
// Cloudflare Worker
addEventListener('fetch', event => {
  const response = event.respondWith(handleRequest(event.request));
  response.headers.set('Access-Control-Allow-Origin', '*');
  return response;
});
```

---

### Download Hangs

- Check file permissions in R2 (must be public)
- Verify file exists at exact URL
- Check firewall/antivirus blocking download

---

### App Won't Install Update

- Windows: Check if antivirus is blocking
- macOS: Check Gatekeeper settings
- Verify SHA512 hash matches
