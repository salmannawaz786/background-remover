# Git Push Guide - Background Remover

## Step 1: Verify Security

✅ .gitignore configured (excludes sensitive files)
✅ .env.example created (template without secrets)
✅ No hardcoded credentials in code
✅ firebase-credentials.json excluded
✅ .env excluded

## Step 2: Initialize Git (if not already done)

cd d:\background-remover-master
git init

## Step 3: Add Remote Repository

git remote add origin https://github.com/Johny111ishxb/background-remover.git

# Or if already exists:
git remote set-url origin https://github.com/Johny111ishxb/background-remover.git

## Step 4: Stage Files

# Add all files (respects .gitignore)
git add .

# Check what will be committed
git status

## Step 5: Commit Changes

git commit -m "Initial commit: Smart background removal with RVM + RMBG"

## Step 6: Push to GitHub

# First push (set upstream)
git push -u origin main

# Or if branch is master:
git push -u origin master

## Troubleshooting

### If branch doesn't exist:
git branch -M main
git push -u origin main

### If authentication fails:
# Use GitHub Personal Access Token instead of password
# Generate at: https://github.com/settings/tokens

### If repository is private and you want to make it public:
# Go to: https://github.com/Johny111ishxb/background-remover/settings
# Scroll to "Danger Zone" → "Change visibility" → "Make public"

## Files That Will Be Pushed

✅ Source code (*.py, *.js, *.html, *.css)
✅ Static assets (images, icons)
✅ Templates
✅ Requirements.txt
✅ README.md
✅ .env.example
✅ .gitignore
✅ GitHub Actions workflow

## Files That Will NOT Be Pushed (Protected)

❌ .env (your secrets)
❌ firebase-credentials.json
❌ *.onnx (model files - too large)
❌ uploads/ folder
❌ *.log files
❌ __pycache__/
❌ venv/
❌ Documentation files (DEPLOYMENT_GUIDE.md, etc.)
❌ models.py (example file)
❌ Untitled-*.txt

## After Pushing

1. Verify files on GitHub: https://github.com/Johny111ishxb/background-remover
2. Check that .env and firebase-credentials.json are NOT visible
3. Add repository description and topics
4. Enable GitHub Actions if needed

## Security Checklist

✅ No API keys in code
✅ No passwords in code
✅ No Firebase credentials in code
✅ All secrets in .env (not committed)
✅ .env.example provided for others
✅ .gitignore properly configured

You're safe to push! 🚀
